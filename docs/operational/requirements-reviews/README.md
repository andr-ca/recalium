# Requirements Reviews

This folder stores requirements review inputs and outputs.

## Artifact types
- [recalium-v1-review-handoff.md](recalium-v1-review-handoff.md) — current handoff package for requirements review
- [requirements-review-checklist-template.md](requirements-review-checklist-template.md) — template for completed requirements review checklists

## Required review output
- Completed requirements reviews should be saved in this folder.
- Preferred filename pattern: `<task-name>-requirements-review-checklist.<YYYYMMDD>T<HHMMSS>Z.md`.
- The completed artifact should be a checklist, not only a prose summary.

## Minimum checklist expectations
- Confirm a canonical high-level requirements index exists when multiple requirements files are present.
- Confirm the high-level requirements index explains the product or feature at a high level.
- Confirm every newly created or updated requirements file is linked from that high-level document.
- Confirm each atomic requirement uses a unique identifier in the form `<feature-short-name>-NNN`.
- Confirm completeness, consistency, edge-case coverage, traceability, and implementation readiness.
