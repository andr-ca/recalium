# Recalium v1.1 Quality Baseline & Flagship Claim

**Status:** measured — v1.1 baseline
**Method:** `make eval` (see [evals/README.md](../../../evals/README.md)) against the
live stack with local CPU embeddings (`all-MiniLM-L6-v2`) and Ollama `qwen3.5:4b`
for extraction.
**Evidence:** [eval-v1.1-2026-07-09/](../tests/artifacts/eval-v1.1-2026-07-09/)
(`report.md` + `results.json`, 5/5 checks passed) plus prior timestamped runs
under [artifacts/](../tests/artifacts/).

This converts Recalium's central, previously-unmeasured claim ("retrieve relevant,
source-backed context") into a **measured and transparent** one.

## Flagship claim

> **On the v1.1 golden set, Recalium's hybrid retrieval reaches 100% recall@10
> and finds paraphrases keyword search misses (semantic lift +100%); extracted
> facts carry verbatim source spans (span fidelity 100% on the extraction set).**

> **Caveat (GPT5.6 #3/#20):** the golden set is small (4 conversations, 13
> queries) and was also used for tuning, and recall is scored at archive level.
> Treat these as **development smoke metrics**, not release-grade evidence. A
> sealed holdout + real-corpus run (200 anonymized exports) with strict/release
> mode is required before any public quality claim.

### Definitions

- **recall@10** — fraction of relevant golden items present in the top 10 results.
- **hybrid** — Reciprocal Rank Fusion of keyword (FTS) + semantic (pgvector),
  see [retrieval-and-ranking.md](../../architecture/retrieval-and-ranking.md).
- **semantic lift** — paraphrase recall@10(semantic) − recall@10(keyword) on
  `semantic_only` queries (paraphrases sharing no content-word stems).
- **span fidelity** — fraction of extracted facts whose `source_span` is a
  verbatim substring of the raw source (hardened at write time by F4).

## Limitations (transparency)

- Golden set is **synthetic** AI-conversation data; no multi-turn reasoning
  validation. Numbers are a floor for representative content, not a universal
  guarantee.
- Extraction figures use a local 4B model (`qwen3.5:4b`); a larger BYOK provider
  typically improves recall/precision.
- `semantic`/paraphrase metrics require embeddings (`EMBED_BACKEND=cpu`); in
  degraded mode retrieval falls back to keyword and paraphrase scoring is skipped.

## Reproduce

```bash
make eval   # requires the stack running; writes report.md + results.json
```

Add `evals/` to pre-release CI so every candidate build re-establishes this
baseline and regressions are caught before release.
