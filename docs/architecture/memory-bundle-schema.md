# Memory Bundle Schema (v1)

**Status:** stable — v1 contract
**Format id:** `recalium-memory-bundle`
**Producer/consumer:** `GET /api/export/bundle` · `POST /api/import/bundle`
([backend/app/api/routes/portability.py](../../backend/app/api/routes/portability.py))

This is the formal, versioned specification of the JSON format Recalium uses to
export and import memory. It substantiates the "open format" positioning claim
and lets external tools read/write Recalium memory without reverse-engineering
the export code.

> **Scope (GPT5.6 #8):** v1 is a **source-archive bundle** — it carries raw
> archive items only. It does **not** include derived memory (summaries, facts,
> links, embeddings), canonical items, stable derived IDs, or audit history;
> those are regenerated on import. A v1 round-trip is therefore **not** lossless
> "portable memory." A versioned full-graph bundle is planned — see
> [gpt5.6-sol-recommendations-status.md](../gpt5.6-sol-recommendations-status.md).

## Design goals

- **Portable & inspectable** — plain JSON, human-readable, no binary blobs.
- **Provenance-preserving** — every item keeps its source identity and hash.
- **Idempotent import** — re-importing a bundle is safe (content-hash dedup).
- **Forward-compatible** — versioned; unknown fields are ignored by importers.

## Top-level shape

```json
{
  "format": "recalium-memory-bundle",
  "version": "1",
  "exported_at": "2026-07-09T11:55:53.616903+00:00",
  "items": [ { "...": "see item schema" } ]
}
```

- `format` — MUST equal `recalium-memory-bundle`. Import rejects other values (HTTP 422).
- `version` — bundle contract version (currently `"1"`, a string). Import rejects mismatches (HTTP 422).
- `exported_at` — ISO 8601 UTC timestamp when the bundle was produced.
- `items` — array of archive items (may be empty). Deleted items are excluded (PRIV-03).

## Item schema

| Field | Type | Notes |
| --- | --- | --- |
| `id` | string (UUID) | Source archive item id. Informational on import (new ids are assigned). |
| `source_type` | string | e.g. `chatgpt`, `claude`, `mcp`, `file`. |
| `source_name` | string \| null | Human label / conversation or session name. |
| `ingested_at` | string (ISO 8601) | When the item was originally ingested. |
| `raw_content` | string | **Required on import.** The raw conversation/text. |
| `content_hash` | string | Dedup key. Import skips items whose hash already exists (including deleted). |
| `conversation_count` | integer | Number of conversations/messages represented. |
| `metadata` | object \| null | Free-form source metadata (client identity, ids, URIs, hints). |

Derived memory (facts, summaries, embeddings, links, canonical items) is **not**
included in v1 — it is regenerated by the processing pipeline on import from
`raw_content`. This keeps the bundle small and provider-agnostic, and guarantees
derived memory matches the importer's own models/config.

## JSON Schema (Draft 2020-12)

```json
{
  "$schema": "https://json-schema.org/draft/2020-12/schema",
  "$id": "https://recalium.local/schemas/memory-bundle/v1.json",
  "title": "Recalium Memory Bundle",
  "type": "object",
  "required": ["format", "version", "items"],
  "properties": {
    "format": { "const": "recalium-memory-bundle" },
    "version": { "const": "1" },
    "exported_at": { "type": "string", "format": "date-time" },
    "items": {
      "type": "array",
      "items": {
        "type": "object",
        "required": ["raw_content"],
        "properties": {
          "id": { "type": "string" },
          "source_type": { "type": "string" },
          "source_name": { "type": ["string", "null"] },
          "ingested_at": { "type": "string", "format": "date-time" },
          "raw_content": { "type": "string", "minLength": 1 },
          "content_hash": { "type": "string" },
          "conversation_count": { "type": "integer", "minimum": 0 },
          "metadata": { "type": ["object", "null"] }
        },
        "additionalProperties": true
      }
    }
  },
  "additionalProperties": true
}
```

`additionalProperties: true` is intentional: importers ignore unknown fields so
future minor additions don't break older readers.

## Import semantics

1. Validate `format` and `version` (HTTP 422 on mismatch).
2. `items` must be a list (HTTP 422 otherwise).
3. For each item: require non-empty `raw_content`; if `content_hash` already
   exists (including deleted items), **skip** (idempotent). Otherwise ingest.
4. Response: `{"imported": N, "skipped": M, "errors": [...]}`.

## Versioning & evolution

- The bundle `version` is independent of the HTTP API version (`X-API-Version`).
- **Backward-compatible** changes (new optional item fields) keep `version = "1"`.
- **Breaking** changes (renamed/removed fields, changed semantics) bump to `"2"`
  and add a migration note here. Importers reject unknown major versions.

## Extension points

- `metadata` — attach custom source metadata without a schema change.
- Additional top-level keys — permitted and ignored by v1 importers; document
  any you rely on and gate behavior on their presence.

## Example (minimal)

```json
{
  "format": "recalium-memory-bundle",
  "version": "1",
  "exported_at": "2026-07-09T11:55:53.616903+00:00",
  "items": [
    {
      "id": "5933f174-0375-4917-af40-88c4596d5f6c",
      "source_type": "chatgpt",
      "source_name": "Async patterns in Python",
      "ingested_at": "2026-07-08T07:32:00+00:00",
      "raw_content": "User: How do I use asyncio.gather? ...",
      "content_hash": "9f2b...",
      "conversation_count": 1,
      "metadata": { "conversation_id": "abc123" }
    }
  ]
}
```
