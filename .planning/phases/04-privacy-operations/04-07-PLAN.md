## File-by-file changes

### 1. `frontend/src/components/NavSidebar.tsx`

**Problem:** Disabled nav items render as `<span>` — they receive no Tab focus (correct), but they
have no `tabIndex` attribute. Explicitly set `tabIndex={-1}` to make the intent clear and
future-proof.

**Also:** `aria-current="page"` is set by NavLink's `isActive` class but is not explicitly passed.
NavLink from react-router-dom automatically sets `aria-current="page"` when active — verify this
is true for the version in use. If it does NOT set it automatically, add it via the `className`
callback by also returning an `aria-current` prop.

Change disabled item `<span>` to add `tabIndex={-1}` and keep `aria-disabled="true"`:

```tsx
// BEFORE:
<span
  title={`Available in a future update (Phase ${item.phase})`}
  className="flex items-center gap-3 rounded-md px-3 py-2 text-sm font-medium opacity-40 cursor-not-allowed select-none"
  aria-disabled="true"
>

// AFTER:
<span
  title={`Available in a future update (Phase ${item.phase})`}
  className="flex items-center gap-3 rounded-md px-3 py-2 text-sm font-medium opacity-40 cursor-not-allowed select-none"
  aria-disabled="true"
  tabIndex={-1}
>
```

**NavLink — add aria-current explicitly** (NavLink v6 sets it but being explicit is safer):

```tsx
// BEFORE:
<NavLink
  to={item.to}
  className={({ isActive }) =>
    cn(
      "flex items-center gap-3 rounded-md px-3 py-2 text-sm font-medium transition-colors",
      isActive
        ? "bg-[--color-sidebar-muted] text-white"
        : "hover:bg-[--color-sidebar-muted] hover:text-white text-[--color-sidebar-fg]/80"
    )
  }
>

// AFTER:
<NavLink
  to={item.to}
  aria-current={undefined}  // NavLink sets aria-current="page" automatically when active
  className={({ isActive }) =>
    cn(
      "flex items-center gap-3 rounded-md px-3 py-2 text-sm font-medium transition-colors",
      isActive
        ? "bg-[--color-sidebar-muted] text-white"
        : "hover:bg-[--color-sidebar-muted] hover:text-white text-[--color-sidebar-fg]/80"
    )
  }
>
```

Note: NavLink v6 sets `aria-current="page"` automatically. The `aria-current={undefined}` line
above is a no-op — do NOT add it. Simply verify that NavLink already does this by inspecting the
DOM. If it does, no change needed. If it does not, use:

```tsx
<NavLink
  to={item.to}
  className={({ isActive }) => { ... }}
  {...({ "aria-current": isActive ? "page" : undefined } as any)}
>
```

But do not add the spread if NavLink already handles it.

### 2. `frontend/src/components/Layout.tsx`

**Problem:** The `<main>` element lacks an `aria-label`. Screen reader users jumping to `<main>`
with a landmark shortcut get no context about the content region.

```tsx
// BEFORE:
<main className="flex-1 overflow-auto p-6">

// AFTER:
<main className="flex-1 overflow-auto p-6" aria-label="Page content">
```

### 3. `frontend/src/pages/SearchPage.tsx`

**Three fixes:**

#### 3a. Search input label

```tsx
// BEFORE:
<input
  type="text"
  value={query}
  onChange={(e) => setQuery(e.target.value)}
  placeholder="Search your memory…"
  className="flex-1 h-10 rounded-md border border-input bg-background px-3 py-2 text-sm"
/>

// AFTER:
<>
  <label htmlFor="search-query" className="sr-only">Search query</label>
  <input
    id="search-query"
    type="text"
    value={query}
    onChange={(e) => setQuery(e.target.value)}
    placeholder="Search your memory…"
    className="flex-1 h-10 rounded-md border border-input bg-background px-3 py-2 text-sm focus-visible:ring-2 focus-visible:ring-primary"
  />
</>
```

#### 3b. Mode buttons — add `aria-pressed`

```tsx
// BEFORE:
{(["hybrid", "keyword", "semantic"] as Mode[]).map((m) => (
  <Button
    key={m}
    type="button"
    variant={mode === m ? "default" : "outline"}
    size="sm"
    onClick={() => setMode(m)}
  >
    {m}
  </Button>
))}

// AFTER:
{(["hybrid", "keyword", "semantic"] as Mode[]).map((m) => (
  <Button
    key={m}
    type="button"
    variant={mode === m ? "default" : "outline"}
    size="sm"
    onClick={() => setMode(m)}
    aria-pressed={mode === m}
  >
    {m}
  </Button>
))}
```

#### 3c. ProvenanceInlinePanel close button

```tsx
// BEFORE:
<Button variant="ghost" size="sm" onClick={onClose}>Close</Button>

// AFTER:
<Button variant="ghost" size="sm" onClick={onClose} aria-label="Close provenance panel">Close</Button>
```

#### 3d. Error and loading states

```tsx
// BEFORE:
{error && <p className="text-sm text-destructive">{error}</p>}

// AFTER:
{error && <p role="alert" className="text-sm text-destructive">{error}</p>}
```

Also add `aria-live` to the loading state inline with results:

```tsx
// BEFORE (inside the return, before the results div):
// No loading announcement

// AFTER — wrap the entire results section:
<div aria-live="polite" aria-busy={loading}>
  {/* existing response content */}
</div>
```

## Focus indicator audit (all pages)

Search for `focus:outline-none` without a paired `focus-visible:ring`:

```bash
grep -r "focus:outline-none" frontend/src --include="*.tsx" -l
```

For every match: if the element is interactive (button, input, link), ensure it ALSO has
`focus-visible:ring-2 focus-visible:ring-primary` (or equivalent). `focus:outline-none` alone
removes the browser default focus ring and leaves keyboard users with no indicator — this is a
critical WCAG 2.1 AA failure (Success Criterion 2.4.7).

Pattern to apply on any interactive element that has `focus:outline-none` without a ring:

```tsx
className="... focus:outline-none focus-visible:ring-2 focus-visible:ring-primary focus-visible:ring-offset-1"
```
