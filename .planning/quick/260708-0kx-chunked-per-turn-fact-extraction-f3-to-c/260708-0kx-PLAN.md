# Quick Task 260708-0kx: Chunked per-turn fact extraction (F3)

**Mode:** quick (inline; TDD)
**Created:** 2026-07-08

## Why

Eval run 3 measured extraction at recall 57.7% / precision 65.6% (thresholds
60% / 70%) with qwen3.5:4b. Root cause (verified against extracted facts):
the model extracts accurately but predominantly from the FIRST turn of
multi-turn conversations — conv-001's four missed facts all live in the second
assistant reply. This is F3 from docs/recommendations.md (also covers silent
truncation of very long conversations).

## Tasks

1. `_split_conversation(text, max_chunk_chars=1200)` — split at turn
   boundaries (User:/Assistant:/Human:/AI:), greedy packing, hard-split
   oversized turns; every chunk is a contiguous verbatim slice so
   source_spans stay verbatim in the full document. TDD (4 unit tests).
2. `_run_extract_job` — extract per chunk (all providers), merge,
   `_dedupe_facts` by normalized fact_text (chunk-overlap duplicates).
3. Rebuild, re-run `make eval` with Ollama; extraction must clear
   recall ≥ 60% / precision ≥ 70% (span fidelity / provenance stay 100%).
